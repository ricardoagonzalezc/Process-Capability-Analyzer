The purpose of this project is to understand and experiment with key Statistical Process Control (SPC) definition, gaining experience with tools that are frequently used in high volume manufacturing environments and are essential in the implementation of Lean Six Sigma tools. 

Below you will find the definitions for each concept:

Process Capability (Cp): measures the potential capability of a process, assuming it is perfectly centered between specification limits. It only looks at the spread of the process (Six sigma) relative to the tolerance range. 

Cp=0.7 Not capable
Cp=1.0 Barely capable
Cp=2.0 Capable

Process Capability Index (Cpk): measures the actual capability of a process, accounting for how far the mean has shifted from the center. It considers both the spread and centereing, using within-subgroup variation (short term). 

Cpk<=0.7 Not capable
Cpk<=1.0 Barely capable
Cpk<=2.0 Capable

Process Performance (Pp): measures the potential performance of a new process or a process not yet in statistical control. It uses the total/overall standard deviation (long term).

Process Performance Index (Ppk): measures the actual long-term performannce, accounting for shifts and drifts in the process over time. It represents how the process has performed in the past. 

Mean: often referred to as the average, it is obtained by adding up all the numbers in the dataset and then dividing the sum by the total number of values in the set. 

Standard Deviation (stdv): it is a measure of dispersion or variability within a dataset. It gives us an idea of how spread out the numbers in a data set are from their mean. A small stdv indicates that the data points are close to the mean, and a large stdv suggest that the data points are spread out over a wider rage.
